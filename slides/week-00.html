Step 1 — your submission repo

  • Account at github.com/signup — professional username, verify your email.
  • 2FA on: Settings → Password and authentication, via an authenticator app (Authy, Google Authenticator, 1Password).
  • github.com/new: name training-ai-fde (lowercase), visibility Public, init with a README.
  • URL: github.com/<your-username>/training-ai-fde — e.g. github.com/akmalakhpah/training-ai-fde
  • Send the program your repo URL in the onboarding form.
How you submit, every week

One repo, one folder per week

  • Folders: week-00, week-01, week-02 … week-12.
  • README is the index — link each week from it.
  • Submit = push the work, then share the link — a file, a folder, a pull request, or a release tag, as each week specifies.
  • Public forces clean commits, readable READMEs, and honest history from day one.
  • Sensitive deliverable? Raise it with the team; do not make the repo private.

Live · install in order

Step 2 — core tools

  1. VS Code → open the integrated terminal (Ctrl+`); add the Claude Code, Python, + (Windows) WSL extensions.
  2. Claude → install Claude Desktop, sign the Desktop app + Claude Code extension into the Pro account, ask it about your repo.
  3. Git → Source Control panel's Install Git prompt, git config --global user.name / user.email, then Accounts → Sign in to GitHub.
  4. Python → open a .py file, let the extension install it; python3 --version (3.12+).
  5. Node → node --version (v20+), npm --version.

Do not leak your key

Secrets stay out of git

  • Claude Code needs no key — it logs in with your Claude Pro account.
  • API-building weeks only: make your own free key at console.anthropic.com, store it in an env var, never in a file.
  • macOS: export ANTHROPIC_API_KEY="..." in ~/.zshrc; verify echo $ANTHROPIC_API_KEY.
  • Windows: System → Environment Variables (or setx).
  • A key committed to a public repo is a setup failure.
  • If it lands in git history: rotate the key immediately, tell the team.

About 1 hour · ungraded

The diagnostic

  • A small coding warm-up: a function plus a couple of test cases.
  • A GitHub operation: branch and commit.
  • A command-line task: navigate, run a script, read an env var.
  • Read an unfamiliar function: what it does, what would break it.
  • Commit to week-00/ and share the link.
